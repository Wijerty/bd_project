# Панель безопасности антифрод системы

Веб-интерфейс для аналитиков безопасности по обнаружению мошеннических P2P переводов.

## Возможности

- Мониторинг транзакций в реальном времени
- Панель подозрительных транзакций
- Выявление клиентов высокого риска
- Анализ паттернов транзакций
- Детальный просмотр транзакций и клиентов
- Возможность помечать транзакции и блокировать клиентов

## Требования

- Python 3.7+
- PostgreSQL база данных с антифрод схемой
- Node.js и npm (для разработки)

## Установка

1. Перейдите в директорию панели безопасности:
   ```
   cd security_dashboard
   ```

2. Создайте виртуальное окружение:
   ```
   python -m venv venv
   ```

3. Активируйте виртуальное окружение:
   - В Windows:
     ```
     venv\Scripts\activate
     ```
   - В macOS/Linux:
     ```
     source venv/bin/activate
     ```

4. Установите необходимые пакеты Python:
   ```
   pip install -r requirements.txt
   ```

5. Настройте переменные окружения:
   ```
   export DB_HOST=localhost
   export DB_NAME=antifraud_p2p
   export DB_USER=antifraud_user
   export DB_PASSWORD=antifraud_pass
   export DB_PORT=5432
   ```

## Запуск приложения

1. Запустите Flask приложение:
   ```
   python app.py
   ```

2. Откройте браузер и перейдите по адресу:
   ```
   http://localhost:5000
   ```

## API endpoints

- `GET /` - Главная страница панели
- `GET /api/transactions` - Последние транзакции
- `GET /api/flagged-transactions` - Помеченные транзакции
- `GET /api/high-risk-clients` - Клиенты высокого риска
- `GET /api/transaction-patterns` - Паттерны транзакций
- `GET /api/client/<id>` - Детали клиента
- `GET /api/transaction/<id>` - Детали транзакции
- `POST /api/flag-transaction` - Пометить транзакцию
- `POST /api/block-client` - Заблокировать клиента

## Разработка

Для изменения панели:

1. HTML шаблоны находятся в директории `templates/`
2. CSS стили в файле `static/css/style.css`
3. JavaScript функциональность в файле `static/js/dashboard.js`
4. API endpoints определены в файле `app.py`

## Меры безопасности

- В рабочей среде обеспечьте надлежащую аутентификацию и авторизацию
- Используйте HTTPS для всех соединений
- Реализуйте ограничение частоты запросов к API endpoints
- Очищайте все пользовательские вводы
- Регулярно обновляйте зависимости

## Устранение неполадок

- Убедитесь, что база данных запущена и доступна
- Проверьте правильность установки переменных окружения
- Убедитесь, что зависимости Python установлены
- Проверьте консоль браузера на наличие ошибок JavaScript
